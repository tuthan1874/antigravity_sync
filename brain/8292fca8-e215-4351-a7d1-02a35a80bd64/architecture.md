| Table | SELECT | INSERT | UPDATE | DELETE |
|-------|--------|--------|--------|--------|
| `profiles` | Own profile only | âŒ (trigger only) | Own profile only | âŒ |

| `members` | Same workspace | Self-join as guest | Admin only (role change) | Admin only |

### 4.3 Key Security Invariants

| Invariant | Enforcement |
|-----------|-------------|
| **User cannot see other workspaces' data** | RLS filters by `workspace_id` membership |
| **Guest/Member cannot see unassigned tasks** | RLS checks `assignee_id = auth.uid()` OR admin/manager role |
| **Profile creation is server-only** | No INSERT policy on `profiles`, trigger on `auth.users` |
| **Activity logs are immutable** | No UPDATE/DELETE policies, trigger-only INSERT |
| **Role changes require admin** | RLS UPDATE policy checks `role = 'admin'` |
| **File access requires entity access** | Edge Function validates before presigned URL |
| **Soft-deleted data is hidden by default** | All SELECT policies include `deleted_at IS NULL` |
| **n8n has internal-only access** | Webhook secret + internal network |

## 5. Realtime Subscription Security

```
Frontend subscribes to:
  - tasks (filtered by RLS)
  - comments (filtered by RLS)
  - activity_logs (filtered by RLS)

Supabase Realtime RESPECTS RLS:
  - User only receives events for rows they can SELECT
  - No additional filtering needed in frontend
  - Subscription automatically scoped to user's access
```
